# Codex 输出审查清单

每次 Codex 完成任务后，先按以下清单检查。

## 1. 文件层面

- [ ] Codex 是否说明修改了哪些文件？
- [ ] 是否创建了不必要的新目录？
- [ ] 是否删除了已有重要文件？
- [ ] 是否修改了与任务无关的大量代码？
- [ ] 是否引入了不必要依赖？

## 2. 功能层面

- [ ] 本轮目标是否完成？
- [ ] 验收路径是否可操作？
- [ ] 页面是否可以访问？
- [ ] 是否破坏已有功能？
- [ ] 是否偏离“虚拟实验”方向？

## 3. 代码层面

- [ ] 组件命名是否清晰？
- [ ] 路由是否正确？
- [ ] 状态管理是否清楚？
- [ ] 是否有明显重复代码？
- [ ] 是否有硬编码过多的问题？
- [ ] 是否有未处理异常？

## 4. 运行层面

- [ ] 前端能启动。
- [ ] 后端能启动。
- [ ] 控制台无致命错误。
- [ ] 页面刷新不白屏。
- [ ] 主要按钮能点击。

## 5. 方向层面

- [ ] 是否仍围绕实验目标？
- [ ] 是否体现实验变量？
- [ ] 是否体现结果分析？
- [ ] 是否为报告生成做准备？
- [ ] 是否避免了评论、收藏、后台等非核心功能？

## 6. 给 GPT 的反馈格式

验收后建议按以下格式反馈：

```text
本轮 Codex 输出验收：

1. 能否启动：是 / 否
2. 已通过的内容：
- ...

3. 发现的问题：
- 问题 1：...
- 问题 2：...

4. 控制台或终端报错：
粘贴日志

5. 我希望下一步：
- 修复问题 / 继续下一阶段 / 调整页面方向
```
