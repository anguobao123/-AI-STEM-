# 10 测试与验收文档

## 1. 验收原则

本项目的验收重点不是功能数量，而是虚拟实验闭环是否成立。

核心验收问题：

1. 用户是否知道实验目标？
2. 用户是否能操作实验变量？
3. 用户是否能训练模型？
4. 用户是否能观察训练过程？
5. 用户是否能测试模型？
6. 用户是否能分析错误？
7. 用户是否能生成报告？

## 2. 页面验收

| 页面 | 验收标准 |
|---|---|
| 首页 | 能清楚说明项目是什么，有开始实验入口 |
| 导学页 | 能解释核心概念，有进入实验入口 |
| 实验选择页 | 能看到 5 个实验任务 |
| 实验操作页 | 能按步骤完成上传、配置、训练、测试 |
| 分析页 | 有指标、曲线、混淆矩阵、错误样本 |
| 报告页 | 能生成结构完整报告 |
| 展示页 | 能看到已完成作品 |

## 3. 功能验收

### 图片上传

1. 支持 jpg/png/webp。
2. 能按类别显示图片。
3. 能统计每类样本数量。
4. 样本不足时有提示。

### 训练参数

1. 能设置 epochs。
2. 能设置 learning rate。
3. 能设置 batch size。
4. 能开启或关闭数据增强。
5. 有推荐配置。

### 模型训练

1. 能加载 TensorFlow.js 模型。
2. 能启动训练。
3. 能显示当前 epoch。
4. 能更新 loss / accuracy 曲线。
5. 训练完成后有结果摘要。

### 模型测试

1. 能上传测试图片。
2. 能显示预测类别。
3. 能显示置信度。
4. 能标记真实类别。
5. 能记录错误样本。

### 结果分析

1. 能显示训练准确率。
2. 能显示验证准确率。
3. 能显示训练曲线。
4. 能显示混淆矩阵。
5. 能展示错误样本。
6. 能给出改进建议。

### 报告生成

1. 报告包含实验目的。
2. 报告包含实验原理。
3. 报告包含数据集说明。
4. 报告包含参数设置。
5. 报告包含结果指标。
6. 报告包含错误分析。
7. 报告包含实验结论。
8. 报告内容可复制。

## 4. 演示验收

5 到 10 分钟演示流程：

1. 进入首页。
2. 说明实验目标。
3. 选择样本数量实验。
4. 上传或使用示例数据集。
5. 设置训练轮次。
6. 启动训练。
7. 查看曲线。
8. 上传测试图片。
9. 查看混淆矩阵和错误样本。
10. 生成报告。

## 5. 稳定性验收

1. 使用 Chrome 或 Edge 能正常打开。
2. 每类 10 张图片、2 类分类可完成训练。
3. 页面刷新后不出现白屏。
4. 控制台无持续红色报错。
5. 训练失败时有提示。
6. 上传异常图片时有提示。

## 6. 不通过标准

出现以下情况视为不通过：

1. 只能点击按钮，没有实验变量。
2. 不能完成模型训练。
3. 没有结果分析。
4. 没有实验报告。
5. 页面主线像普通管理平台，不像虚拟实验。
6. 大量功能不可用或只是空壳。
