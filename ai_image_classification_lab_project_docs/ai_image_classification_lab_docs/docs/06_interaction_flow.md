# 06 交互流程设计文档

## 1. 用户完整流程

1. 用户进入首页。
2. 查看项目介绍。
3. 进入实验导学。
4. 选择实验任务。
5. 创建类别。
6. 上传图片。
7. 设置训练参数。
8. 启动训练。
9. 查看训练曲线。
10. 上传测试图片。
11. 查看预测结果。
12. 查看混淆矩阵。
13. 查看错误样本分析。
14. 生成实验报告。
15. 发布或保存作品。

## 2. 图片上传流程

1. 用户选择类别。
2. 点击上传图片。
3. 系统检查格式。
4. 系统生成预览。
5. 系统计入类别样本数量。
6. 如果图片数量过少，给出提醒。
7. 如果格式错误，拒绝上传并提示。

支持格式：jpg、jpeg、png、webp。

## 3. 训练参数设置流程

1. 用户选择训练轮次。
2. 用户设置学习率或使用推荐配置。
3. 用户设置 batch size。
4. 用户选择是否开启数据增强。
5. 系统显示当前变量说明。
6. 用户点击开始训练。

推荐配置：

- epochs：20
- learningRate：0.001
- batchSize：16
- validationSplit：0.2
- augmentation：默认关闭

## 4. 训练流程

1. 系统加载 MobileNet。
2. 系统预处理用户图片。
3. 系统提取特征向量。
4. 系统训练分类头。
5. 每个 epoch 更新曲线。
6. 训练完成后生成指标。
7. 用户进入测试步骤。

## 5. 测试流程

1. 用户上传测试图片。
2. 系统预处理图片。
3. 模型输出预测类别。
4. 系统显示 Top 3 结果。
5. 用户标记真实类别。
6. 系统判断是否预测正确。
7. 如果错误，加入错误样本列表。

## 6. 结果分析流程

1. 系统汇总训练参数。
2. 系统汇总训练指标。
3. 系统渲染训练曲线。
4. 系统计算混淆矩阵。
5. 系统展示错误样本。
6. 系统生成改进建议。

## 7. 报告生成流程

1. 用户点击生成报告。
2. 系统读取实验记录。
3. 系统填充报告模板。
4. 系统插入图表和指标。
5. 用户可编辑结论。
6. 用户保存、复制或导出。

## 8. 异常处理流程

### 图片数量不足

提示：当前类别样本较少，可以继续实验，但结果可能不稳定。

### 模型加载失败

提示：模型资源加载失败，请检查网络或刷新页面。

### 训练速度过慢

提示：浏览器端训练受设备性能影响，可减少图片数量或训练轮次。

### 浏览器不支持

提示：当前浏览器可能不支持 WebGL，请使用最新版 Chrome 或 Edge。
