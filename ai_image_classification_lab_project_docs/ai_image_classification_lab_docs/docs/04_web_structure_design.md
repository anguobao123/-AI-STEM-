# 04 网页结构设计文档

## 1. 路由总览

| 路径 | 页面 | 说明 |
|---|---|---|
| `/` | 首页 | 项目介绍与入口 |
| `/guide` | 实验导学页 | 基础概念学习 |
| `/experiments` | 实验任务选择页 | 选择实验任务 |
| `/lab/:experimentId` | 实验操作页 | 上传、配置、训练、测试 |
| `/analysis/:recordId` | 结果分析页 | 曲线、矩阵、错误样本 |
| `/report/:recordId` | 实验报告页 | 自动生成报告 |
| `/gallery` | 作品展示页 | 展示完成作品 |

## 2. 页面跳转关系

首页 → 实验导学页 → 实验任务选择页 → 实验操作页 → 结果分析页 → 实验报告页 → 作品展示页

用户也可以从导航直接进入实验任务选择页和作品展示页。

## 3. 全局导航结构

导航项：

1. 首页
2. 实验导学
3. 开始实验
4. 作品展示

右侧可保留“我的实验记录”入口，初版可以先用本地模拟或简单页面。

## 4. 页面布局原则

1. 页面主线要清晰，避免复杂平台感。
2. 每个页面只解决一个核心问题。
3. 实验操作页采用步骤向导。
4. 结果分析页强调图表和解释。
5. 报告页强调可提交成果。

## 5. 核心组件清单

| 组件 | 用途 |
|---|---|
| AppHeader | 顶部导航 |
| ExperimentCard | 实验任务卡片 |
| ConceptBlock | 导学知识块 |
| ExperimentStep | 步骤向导 |
| ImageUploader | 图片上传 |
| DatasetSummary | 数据集统计 |
| TrainConfigPanel | 训练参数配置 |
| TrainChart | 训练曲线 |
| PredictionPanel | 推理结果展示 |
| ConfusionMatrix | 混淆矩阵 |
| ErrorSampleList | 错误样本列表 |
| ReportPreview | 报告预览 |
| MetricCard | 指标卡片 |

## 6. 状态设计

### 空状态

1. 未上传图片：显示上传引导。
2. 未开始训练：显示训练参数说明。
3. 无错误样本：显示“当前测试样本未发现错误”。
4. 无实验记录：显示“开始一次实验”。

### 加载状态

1. 模型加载中。
2. 图片处理中。
3. 训练进行中。
4. 报告生成中。

### 错误状态

1. 图片格式不支持。
2. 每类图片数量不足。
3. 模型加载失败。
4. 浏览器不支持 WebGL。
5. 训练过程中断。

## 7. 响应式设计

优先适配桌面端展示。移动端只需保证页面可阅读，不要求完整训练体验。训练页建议最低宽度 1200px。
