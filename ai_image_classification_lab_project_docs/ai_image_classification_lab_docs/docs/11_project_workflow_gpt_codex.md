# 11 GPT + Codex + 人工验收工作流

## 1. 工作模式定义

本项目采用三方协作方式：

1. GPT：需求整合、任务拆解、提示词生成、结果复盘。
2. Codex：根据提示词进行代码实现、修改、修复。
3. 用户：运行项目、页面验收、反馈问题。

## 2. 核心原则

1. 每次只让 Codex 做一个明确阶段任务。
2. 不一次性让 Codex 写完整系统。
3. 每轮任务必须有验收标准。
4. Codex 输出后先运行，再反馈问题。
5. 发现偏离方向时，优先修正主线，不继续堆功能。
6. 所有新功能必须服务虚拟实验闭环。

## 3. 推荐开发阶段

### 阶段 0：项目初始化

目标：创建项目目录、前后端基础结构、运行说明。

Codex 任务：

1. 创建 Vue3 + Vite 前端。
2. 创建 Flask 后端。
3. 添加 README。
4. 添加基础路由。
5. 确保前后端可以分别启动。

验收标准：

1. `npm install` 可成功。
2. `npm run dev` 可打开前端页面。
3. `python app.py` 可启动后端。
4. 首页可访问。

### 阶段 1：前端静态原型

目标：先把页面和流程做出来，不接真实训练。

Codex 任务：

1. 完成首页。
2. 完成导学页。
3. 完成实验选择页。
4. 完成实验操作页步骤向导。
5. 完成结果分析页静态图表。
6. 完成报告页静态模板。
7. 完成作品展示页。

验收标准：

1. 路由完整。
2. 页面不白屏。
3. 页面主线清楚。
4. 能从首页走到报告页。
5. 使用模拟数据展示曲线和矩阵。

### 阶段 2：数据上传与实验配置

目标：实现真实图片上传、类别管理、参数配置。

Codex 任务：

1. 实现类别创建。
2. 实现图片上传和预览。
3. 实现每类样本统计。
4. 实现样本不足提示。
5. 实现训练参数设置。
6. 保存当前实验配置到 Pinia。

验收标准：

1. 能创建 2 到 5 个类别。
2. 每个类别能上传多张图片。
3. 能显示图片缩略图。
4. 能统计样本数量。
5. 参数配置能保存。

### 阶段 3：TensorFlow.js 训练功能

目标：接入真实模型训练。

Codex 任务：

1. 安装 TensorFlow.js。
2. 封装 MobileNet 加载方法。
3. 实现图片预处理。
4. 实现分类头训练。
5. 实现训练回调。
6. 更新训练曲线。
7. 输出训练结果。

验收标准：

1. 能加载模型。
2. 能用 2 类图片训练。
3. 训练过程中曲线更新。
4. 训练完成后有准确率结果。
5. 控制台无致命错误。

### 阶段 4：推理与错误样本分析

目标：完成测试与结果分析。

Codex 任务：

1. 实现测试图片上传。
2. 实现模型推理。
3. 显示 Top 3 置信度。
4. 支持用户选择真实类别。
5. 记录预测错误样本。
6. 计算混淆矩阵。
7. 生成改进建议。

验收标准：

1. 能上传测试图片。
2. 能输出预测类别。
3. 能显示置信度。
4. 能记录错误样本。
5. 能展示混淆矩阵。

### 阶段 5：后端持久化

目标：保存实验记录、报告和展示作品。

Codex 任务：

1. 实现 Flask API。
2. 建立 SQLite 表。
3. 保存实验记录。
4. 保存报告内容。
5. 保存作品展示信息。
6. 前端接入 API。

验收标准：

1. 刷新后记录不丢失。
2. 能查看历史实验。
3. 能打开报告。
4. 能发布到展示页。

### 阶段 6：报告生成与导出

目标：让报告成为完整成果。

Codex 任务：

1. 根据模板生成报告。
2. 自动填充指标和参数。
3. 插入错误分析。
4. 支持复制 Markdown。
5. 支持导出 PDF，若时间不足可后置。

验收标准：

1. 报告内容完整。
2. 报告可复制。
3. 报告能体现实验变量和结果分析。

### 阶段 7：包装与演示优化

目标：让作品适合提交与答辩。

Codex 任务：

1. 补充演示数据。
2. 优化页面文案。
3. 优化视觉布局。
4. 修复明显 bug。
5. 补充启动说明。

验收标准：

1. 5 到 10 分钟可以完整演示。
2. 页面稳定。
3. 项目主线清楚。
4. README 可指导启动。

## 4. 每轮工作格式

### GPT 输出给 Codex

1. 当前阶段。
2. 目标。
3. 项目上下文。
4. 需要修改的文件。
5. 具体任务。
6. 禁止事项。
7. 验收标准。
8. 输出要求。

### Codex 输出后用户验收

1. 是否能启动。
2. 页面是否正常。
3. 功能是否符合任务。
4. 是否引入无关功能。
5. 是否破坏已有功能。
6. 控制台是否报错。
7. 截图或日志反馈给 GPT。

### GPT 根据反馈生成修复提示词

1. 描述问题。
2. 指定复现步骤。
3. 指定期望结果。
4. 限定修改范围。
5. 要求 Codex 不重构无关模块。

## 5. 风险控制

1. 不要让 Codex 一次生成过大项目。
2. 不要在训练功能完成前做社区和后台。
3. 不要过早引入复杂权限。
4. 不要同时改前端、后端、模型训练三个核心区。
5. 每次修改后必须运行验收。
