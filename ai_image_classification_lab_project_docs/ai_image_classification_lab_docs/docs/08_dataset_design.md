# 08 数据集设计文档

## 1. 数据集目标

数据集设计要服务于虚拟实验，而不是追求大规模训练。初版数据集应小、清晰、可快速训练、便于观察变量影响。

## 2. 支持图片格式

1. jpg
2. jpeg
3. png
4. webp

## 3. 图片数量建议

| 类型 | 建议数量 |
|---|---|
| 最低可运行 | 每类 5 张 |
| 推荐实验 | 每类 10 到 30 张 |
| 浏览器端上限 | 每类 50 张以内 |

## 4. 类别数量建议

MVP 支持 2 到 5 类。

推荐初始任务：2 到 3 类。

## 5. 内置示例数据集

建议准备 3 组：

### 示例一：差异明显类别

类别：猫、汽车、花。

用途：适合新手体验，分类效果较好。

### 示例二：中等相似类别

类别：苹果、梨、桃。

用途：展示类别相似度导致的混淆。

### 示例三：生活物品类别

类别：水杯、书本、键盘。

用途：适合展示自定义图像分类模型。

## 6. 图片质量检查规则

上传后进行基础校验：

1. 文件格式是否合法。
2. 文件大小是否超限。
3. 图片是否能正常读取。
4. 分辨率是否过低。
5. 每类图片数量是否过少。

## 7. 用户提示文案

### 图片数量不足

当前类别样本较少，模型可能无法学习到稳定特征。建议每类至少上传 10 张图片。

### 类别不平衡

不同类别样本数量差异较大，可能导致模型偏向样本更多的类别。

### 图片质量问题

部分图片可能过于模糊、过暗或主体不明显，可能影响训练效果。

## 8. 数据划分

默认训练集 / 验证集比例：8:2。

支持用户调整：

1. 70% / 30%
2. 80% / 20%
3. 90% / 10%

## 9. 数据增强选项

MVP 支持：

1. 水平翻转。
2. 随机裁剪。
3. 亮度变化。
4. 轻微旋转。

初版可以先实现开关和说明，真实增强可以分阶段实现。

## 10. 数据集验收标准

1. 能上传图片。
2. 能按类别统计数量。
3. 能提示样本不足。
4. 能提示类别不平衡。
5. 能参与训练流程。
6. 能在报告中记录样本数量。
