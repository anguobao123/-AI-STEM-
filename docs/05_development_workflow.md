# 开发流程

## 当前阶段

当前阶段为 `v0.2` 产品体验重构，核心目标不是继续修模型持久化，而是完成教学演示闭环。

## 推荐开发顺序

1. 先完成首页、导学页和实验列表页的信息重构
2. 再重构实验工作台三栏布局
3. 保留真实训练、单图预测和批量测试能力
4. 调整分析页、报告页、记录页和成果页的数据呈现
5. 更新 README 和 docs，明确当前版本边界

## 验证要求

- 前端执行 `npm run build`
- 后端执行 Python 语法检查
- 页面至少验证以下路由：
  - `/`
  - `/guide`
  - `/experiments`
  - `/lab/sample-count`
  - `/records`
  - `/gallery`

## 版本边界

本阶段不继续扩展以下方向：

- IndexedDB 模型保存主流程
- 模型加载入口
- 模型上传下载
- 后端模型文件存储
- MobileNet
