# AI 图像分类虚拟实验室

## 项目简介

本项目服务于初中八年级 STEM 校本课程《数据如何教会 AI：图像分类模型实验课》。

它不是普通的 AI 工具体验页面，而是一个用于支撑学生完成完整 STEM 实验流程的 AI 图像分类虚拟实验网站。课程围绕一个核心问题展开：

> 数据数量、数据质量和训练方式，会怎样影响 AI 图像分类模型的识别效果？

学生通过网站完成以下学习任务：

1. 选择实验任务；
2. 填写项目设定；
3. 设计实验变量；
4. 准备训练集和测试集；
5. 训练模型 1.0；
6. 单图预测；
7. 批量测试；
8. 查看准确率、错误案例和混淆矩阵；
9. 填写优化方案；
10. 进入模型 2.0；
11. 对比优化前后结果；
12. 形成实验记录、分析报告和成果展示。

## 课程与 STEM 对应关系

### S 科学探究

学生观察 AI 图像识别中的正确与错误现象，分析样本数量、图片质量、拍摄角度、背景复杂度等因素对模型表现的影响。

### T 技术实现

学生使用浏览器端图像分类实验网站完成图片上传、摄像头采集、模型训练、单图预测和批量测试。

### E 工程优化

学生经历模型 1.0 到模型 2.0 的优化迭代，根据测试结果调整数据或训练方案。

### M 数学分析

学生使用准确率、错误率、每类准确率、混淆矩阵和错误案例解释模型表现。

## 摄像头采集说明

摄像头采集功能仅用于采集水果、文具、书本、水杯等普通物品图片。

禁止采集：

1. 学生人脸；
2. 姓名牌；
3. 证件；
4. 隐私屏幕；
5. 其他敏感个人信息。

本项目不做人脸识别，不做人脸检测，不识别学生身份。

## 当前暂不支持

当前项目不支持以下能力：

1. 不支持后端或云端模型文件持久化；
2. 不支持账号级模型跨设备自动复用；
3. 不支持后端保存模型；
4. 不支持账号系统；
5. 不支持教师后台；
6. 不支持人脸识别；
7. 不支持一键加载数据集；
8. 不支持压缩包数据集自动解析；
9. 不支持生产级模型训练平台能力。

注意：前端已支持在当前浏览器本机保存模型，也支持导出 TensorFlow.js 模型文件并导入测试；这些能力不等同于后端模型托管或账号级跨设备同步。

## 本地启动方式

### 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端开发服务器默认地址：

```txt
http://127.0.0.1:5173
```

`frontend/vite.config.js` 中配置了 `/api` 代理，默认转发到 `http://localhost:5000`。

### 启动后端

```bash
cd backend
python -m pip install -r requirements.txt
python app.py
```

后端默认地址：

```txt
http://127.0.0.1:5000
```

健康检查接口：

```txt
http://127.0.0.1:5000/api/health
```

## 构建方式

## 部署说明

项目后续可部署到阿里云轻量应用服务器。简要部署思路如下：

1. 前端执行 `npm run build` 生成 `dist`；
2. 使用 Nginx 托管前端静态文件；
3. 后端 Flask 服务通过 gunicorn 或其他方式运行；
4. Nginx 将 `/api` 请求反向代理到后端；
5. 域名解析到服务器公网 IP；
6. 建议配置 HTTPS，摄像头功能在正式域名下通常需要 HTTPS 或 `localhost` 环境才能正常调用。

部署时不要在代码或文档中写死域名、服务器 IP、用户名或密码。

## 项目边界

本项目是教学实验网站，不是生产级 AI 模型平台。

项目重点是帮助学生理解：

- 数据如何影响模型表现；
- 为什么 AI 会识别错误；
- 如何用准确率、错误案例和混淆矩阵分析模型；
- 如何通过数据优化模型。
