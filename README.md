# AI 图像分类虚拟实验

面向初中 STEM 课程《数据如何教会 AI：图像分类模型实验课》的浏览器端教学实验网站。

它不是一个普通的 AI 演示页，而是一个帮助学生完整体验“提出实验问题 - 设计变量 - 准备数据 - 训练模型 - 评估结果 - 优化方案 - 输出报告”的学习平台。

> 核心问题：数据数量、数据质量和训练方式，会怎样影响 AI 图像分类模型的识别效果？

## 项目概览

学生可以在网站中完成以下学习任务：

1. 选择实验任务
2. 填写项目设定
3. 设计实验变量
4. 准备训练集和测试集
5. 训练模型 1.0
6. 单图预测
7. 批量测试
8. 查看准确率、错误案例和混淆矩阵
9. 填写优化方案
10. 进入模型 2.0
11. 对比优化前后结果
12. 形成实验记录、分析报告和成果展示

## STEM 对应关系

### S - 科学探究

观察 AI 图像识别中的正确与错误现象，分析样本数量、图片质量、拍摄角度和背景复杂度等因素对结果的影响。

### T - 技术实现

使用浏览器端图像分类实验网站完成图片上传、摄像头采集、模型训练、单图预测和批量测试。

### E - 工程优化

通过模型 1.0 到模型 2.0 的迭代过程，结合测试结果调整数据或训练方案。

### M - 数学分析

使用准确率、错误率、分类准确率、混淆矩阵和错误案例解释模型表现。

## 摄像头采集说明

摄像头采集功能仅用于采集水果、文具、书籍、水杯等普通物品图片。

禁止采集：

- 学生人脸
- 姓名牌
- 证件
- 隐私屏幕
- 其他敏感个人信息

本项目不做人脸识别、不做人脸检测，也不识别学生身份。

## 当前不支持

当前项目不支持以下能力：

1. 后端或云端模型文件持久化
2. 账号级模型跨设备自动复用
3. 后端保存模型
4. 账号系统
5. 教师后台
6. 人脸识别
7. 一键加载数据集
8. 压缩包数据集自动解析
9. 生产级模型训练平台能力

注意：前端支持在当前浏览器本机保存模型，也支持导出 TensorFlow.js 模型文件并导入测试，但这不等同于后端托管或账号级同步。

## 技术栈

- 前端：Vue 3、Vite、Vue Router、Pinia、Element Plus、ECharts、TensorFlow.js、Axios
- 后端：Flask、Flask-Cors、Flask-SQLAlchemy
- 数据库：SQLite

## 本地启动

### 启动前端

```bash
cd frontend
npm install
npm run dev
```

前端默认地址：

```txt
http://127.0.0.1:5173
```

`frontend/vite.config.js` 中配置了 `/api` 代理，默认转发到 `http://localhost:5000`。

### 启动后端

```bash
cd backend
python -m pip install -r requirements.txt
python app.py
```

后端默认地址：

```txt
http://127.0.0.1:5000
```

健康检查接口：

```txt
http://127.0.0.1:5000/api/health
```

## 构建方式

前端生产构建：

```bash
cd frontend
npm run build
```

构建产物位于：

```txt
frontend/dist
```

## 推荐课堂流程

1. 教师准备水果分类入门数据集和校园物品项目数据集
2. 学生进入网站选择实验任务
3. 学生填写项目名称、小组名称、作者昵称
4. 学生填写实验目标、实验假设和变量设计
5. 学生按类别上传训练图片和测试图片
6. 学生也可以使用摄像头采集普通物品图片
7. 学生训练模型 1.0
8. 学生进行单图预测和批量测试
9. 学生查看准确率、错误案例和混淆矩阵
10. 学生填写优化方案
11. 学生进入模型 2.0 并重新训练
12. 学生对比模型 1.0 与模型 2.0
13. 学生完成实验结论、反思和成果展示

## 演示建议

推荐演示顺序：

`首页 -> 选择实验任务 -> 项目设定 -> 变量设计 -> 数据准备 -> 训练模型 1.0 -> 批量测试 -> 混淆矩阵与错误案例 -> 优化方案 -> 进入模型 2.0 -> 对比结果 -> 成果展示`

现场演示建议优先使用预置图片数据集，摄像头采集仅作为辅助演示，避免临场采集图片数量不足影响展示效果。

## 部署说明

后续可部署到轻量云服务器。简要思路如下：

1. 前端执行 `npm run build` 生成 `dist`
2. 使用 Nginx 托管前端静态文件
3. 后端 Flask 服务通过 gunicorn 或其他方式运行
4. Nginx 将 `/api` 请求反向代理到后端
5. 域名解析到服务器公网 IP
6. 建议配置 HTTPS，摄像头功能在正式域名下通常需要 HTTPS 或 `localhost` 环境才能正常调用

部署时不要在代码或文档中写死域名、服务器 IP、用户名或密码。

## 项目边界

本项目是教学实验网站，不是生产级 AI 模型平台。

项目重点是帮助学生理解：

- 数据如何影响模型表现
- 为什么 AI 会识别错误
- 如何用准确率、错误案例和混淆矩阵分析模型
- 如何通过数据优化改进模型

## 主要页面
